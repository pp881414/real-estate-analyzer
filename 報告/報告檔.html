📦 資料載入

讀取 real_estate_market_pro.csv，只使用「實價登錄」來源（成交價才可靠，591 掛牌價偏高）。

🧹 資料清洗：IQR 過濾

每個行政區各自做 P5~P95 過濾，板橋的標準和淡水不同，更合理。

def iqr_filter(g): Q1, Q3 = g['單價'].quantile(0.05), g['單價'].quantile(0.95) return g[(g['單價'] >= Q1) & (g['單價'] <= Q3)]

把原始的「行政區、街道、屋齡、樓層、型態」轉成模型能理解的 22 個數值特徵。詳見「特徵工程」頁籤。

✂️ 分區訓練策略

樣本 ≥ 100 筆的行政區訓練專屬模型（共 18 區），不足的偏遠區使用全局備援模型。

if count >= MIN_SAMPLES: self.models[district] = train_one(df_district) # 不足 → fallback 到 global_model

⚠️ Data Leakage 問題與修正

Target Encoding 若用全部資料計算（包含測試集），模型「偷看」了測試集，評估指標虛高。修正後 MAPE 從 9.6% → 11.4%，才是真實表現。

🔑 突破防爬：CSRF Token

591 有防爬保護，直接打 API 會被擋。需先偽裝成真實瀏覽器進入首頁，從 HTML 中提取隱藏的 CSRF Token，之後每次請求都帶著它。

r = session.get("https://sale.591.com.tw/") soup = BeautifulSoup(r.text, "html.parser") token = soup.find("meta", {"name": "csrf-token"})["content"] headers["X-CSRF-TOKEN"] = token

📡 分頁 API 請求

591 有內部 JSON API，比爬 HTML 穩定。透過 firstRow 參數控制分頁，每頁 30 筆，最多爬 5 頁 = 150 筆。每頁間隔 2 秒防封鎖。

for page in range(5): time.sleep(2) res = session.get(f"...&firstRow={page*30}") if len(house_list) < 30: break

🔧 多層 Fallback 解析

API 欄位不穩定，單價可能存在不同欄位名稱，設計 4 層備援確保取到數值。地址用正規表達式提取路段名稱。

💾 輸出 CSV

去除重複物件（同 houseid），只保留有效單價筆數，儲存為 UTF-8-sig CSV 供 Data_Center.py 整合。
